# Project Direction

This repository is for a course subject built as a series of experiments. Each experiment should stand on its own, while gradually contributing toward one final full-stack project later.

## Current Progress

- Experiment 1 is implemented: Multi-Platform Post Composer.
- Experiment 1 includes post composition, validation rules, live previews, MongoDB persistence, GitHub push, and Vercel deployment.

## Development Approach

- Continue adding experiments incrementally.
- Keep each experiment understandable, demonstrable, and connected to the eventual final project.
- Preserve working experiments while expanding the app.
- Prefer clean full-stack slices: frontend UI, backend/API logic, validation, persistence, and deployment.

## Visual Direction

The frontend should become more beautiful and distinctive as the experiments evolve.

Target style:

- Maximalism + brutalism + spatialism.
- Primary palette: green, white, and black.
- Bold layouts with strong contrast.
- Dense but organized interfaces.
- Expressive visuals without losing usability.
- Experimental, course-project energy rather than a plain template feel.

## Design Reminder

Future UI work should avoid looking generic. Build toward a memorable final-project identity while keeping every experiment usable and presentable.
